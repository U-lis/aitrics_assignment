# 0. Ground Rule

- 모든 python 개발 환경은 [uv](https://docs.astral.sh/uv/) 를 사용한다.
- import 시에는 무조건 파일 상단에 global import 를 사용한다. 
  - circular import, test 시 환경 제어 등의 특수한 상황에서만 제한적으로 local import 를 허락한다.
- 이미 동일한 기능을 가지고 있는 함수나 클래스가 구현되어 있는지 확인한다. 재사용 가능한 경우 작업을 중지하고 사용자에게 발견한 내용을 정리해 보고한다.
- 새 파일 생성 시에는 XP 관점과 현재 프로젝트 구조에 기반해 적절한 위치를 선정해 파일을 생성한다.
- 상수를 만들어야 할 경우 env, enum, hard-coded 중 어떤 방식을 택할 지 사용자의 확인을 받는다. 각종 path 또는 설정의 경우 env를 이용하는 것을 기본으로 한다.
- env 는 .env 파일을 사용하며, 서비스 환경 배포에 도움을 주기 위해 .env.example 을 같이 사용한다.
    - .env 는 절대 직접 수정하면 안 되며, 사용자에게 추가/수정을 요청해야 한다.
    - .env.example 은 환경변수 추가/수정 시 항상 변경사항을 반영해 사용자가 변경되는 값을 누락하는 일이 없도록 한다
